# 三级九等文章生成架构设计

## 三级定义

### 第一级：初级 (HSK 1-3) — 模板填充 + 简单AI
- 以模板填空为主（目前的 `generateFallbackArticle` 逻辑）
- 句子结构简单，平均每句 8-15 字
- 生词密度低：每篇文章新词不超过 3-5 个
- 主题：日常生活、动物、家庭
- 每一等控制句子长度和词汇复杂度

### 第二级：中级 (HSK 4-6) — AI自由生成 + 引导
- 以 DeepSeek prompt 生成为主（目前的 `createDailyArticle` 逻辑）
- 句子结构中等，平均每句 12-25 字
- 生词密度中等：每篇新词 5-8 个
- 主题：冒险、科学探索、人际关系
- 每一等控制情节复杂度和抽象概念密度

### 第三级：高级 (HSK 7-9) — AI自由生成 + 弱字循环
- 以 DeepSeek prompt 生成为主
- 句子结构复杂，可以有从句
- 生词密度高、弱字循环嵌入
- 主题：抽象概念、历史、文化、科学原理
- 每一等控制抽象程度和篇章结构复杂度

## 九等细分

每级 3 等 = 9 个等级，控制 4 个维度：

| 维度 | 初级三等控制 | 中级三等控制 | 高级三等控制 |
|------|-------------|-------------|-------------|
| **生词密度** | 1→2→3→5个/篇 | 5→6→8个/篇 | 8→10→12个/篇 |
| **句子复杂度** | 7→10→15字/句 | 12→18→25字/句 | 20→30→40字/句 |
| **情节复杂度** | 单线→两段→三段 | 三段→四段→多角色 | 多角色→子情节→多视角 |
| **抽象程度** | 具体事物→简单感受→简单道理 | 道理→价值观→社会关系 | 抽象概念→批判思考→深层主题 |

## 实现策略

### 1. 等级参数化

定义一个 `GradeConfig` 接口，按 `(level, subTier)` 生成参数：

```typescript
interface GradeConfig {
  maxNewWords: number;       // 每篇最大新词数
  avgSentenceLength: number; // 目标句长
  plotDepth: number;         // 1=单线 2=两段 3=三段+
  useSubplot: boolean;       // 是否有子情节
  abstractionLevel: number;  // 1-3
  weakCharTarget: number;    // 每篇目标弱字复习数
}
```

### 2. Prompt 模板化

`buildPrompt` 根据 `GradeConfig` 动态生成指令，而不是写死。例如：
- 初等1: "每句话不超过10个字，一天里发生了一件简单的事"
- 高等3: "可以使用复杂句式，包含一个配角推动情节，探讨关于坚持与成长的深层主题"

### 3. 模板分层

目前的 `STORY_TEMPLATES` 5个模板 + `HIGHER_STORY_TEMPLATES` 2个模板太少了。
需要为每一级扩充：

- 初级模板：10+ 个，覆盖日常/动物/家庭/食物
- 中级模板：10+ 个，覆盖冒险/探索/人际关系/自然
- 高级模板：8+ 个，覆盖抽象/历史/文化

### 4. 内容校验

生成后增加校验：
- 统计平均句长，超出 `GradeConfig` 范围则降级使用更简单的模板
- 检查目标生词是否成功融入（覆盖率 < 80% 则标记弱词）
- 检查弱字出现次数是否符合目标

## 建议实施顺序

1. 新增 `GradeConfig` 接口 + 四级维度参数表
2. 修改 `buildPrompt` 接受 `GradeConfig` 动态生成指令
3. 扩展模板库（初级+5个，中级+5个，高级+3个）
4. 修改 `getTodayTask` / `createDailyArticle` 传递等级参数
5. 增加生成后校验逻辑
